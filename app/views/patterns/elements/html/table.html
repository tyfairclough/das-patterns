Responsive tables — card UI

Your tables should work on mobile devices. To utilise the responsive table behaviour use the class .responsive against the <table> element.

The mobile magic sauce in the mark-up is driven by data-label="lorem ipsum" in the html which is then rendered in a psuedo element at small screensizes. Using markup like this: content: attr(data-label)

Additional treatments

If you need to highlight columns use the .highlight class on the cells that need highlighting. The grey on black visual treatment passes WCAG AAA at normal and large text sizes with a contrast ratio of 10.85:1.

For times when a large amount of data is absolutely necessary you can utilise the .small classs to reduce the size of the content. Always try to maintain the default size and in cases where you have multiple tables default to small on all tables if any one table requires it. This creates a consistency to the interface and avoids a dissonance that can sometimes occur when users are presented with data in differing formats.
