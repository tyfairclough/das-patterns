GOV.UK elements Buttons

Use buttons to move though a transaction, aim to use only one button per page.

Button text

Button text should be short and describe the action the button performs. 

<input class="button" type="submit" value="Save and continue">

Start now button

Launch your service with a Start now button.

Start now 
<a class="button button-start" href="#" role="button">Start now</a>

Button alignment

Align the primary action button to the left edge of your form, in the user’s line of sight.

Disabled buttons

  • don’t disable buttons, unless there’s a good reason to
  • if you have to disable buttons, make sure they look like you can’t click them (use 50% opacity)
  • an example of a useful disabled option is a calendar with greyed out days where no bookings are available
  • provide another way for the user to continue, add an error message or text to explain why the button is disabled
HTML 

        <input class="button" type="submit" value="Disabled primary button" disabled="disabled">

Secondary buttons

  • don’t have secondary buttons, unless there’s a good reason to
HTML 

        <input class="button secondary" type="submit" value="Secondary button">

Secondary disabled buttons

  • don’t have secondary disabled buttons, unless there’s a good reason to
HTML 
 
        <input class="button secondary" type="submit" value="Secondary disabled button" disabled="disabled">

Button + Link

  • Use this text links to go somewhere else other than the happy path when adjacent to a button.
Button + text link
HTML 
 
        <a class="button text-link">Button + text link</a>

Interruption card button

This is also a primary action button. It should be used only on solid colour backgrounds and when contrast is high enough to maintain accessibility (see interruption card and confirm pages).

        <div class="flash-card">
	<div class="form-group">

	</div>

</div>
